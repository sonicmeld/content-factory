# CONTENT_MODEL.md

## Purpose

This document defines the core business entities of Content Factory.

It is the source of truth for understanding how content flows through the system.

---

# Core Principle

Content Factory is NOT an AI content generation platform.

Content Factory consumes content generated by external systems (9Router AI Infrastructure) and manages:

* Content organization
* Channel management
* Upload scheduling
* YouTube distribution

---

# Primary Entity

## Content Package

The primary entity in Content Factory is a Content Package.

A Content Package represents a complete YouTube upload unit.

Example:

Sleep Sounds #001

* video.mp4
* thumbnail.webp
* description.txt
* timestamp.txt

A Content Package can be assigned to:

* One YouTube Channel
* One Upload Schedule
* One Upload Queue Entry

---

# Supporting Entities

## Channel

Represents a YouTube channel.

Contains:

* Channel Name
* Slug
* OAuth Credentials
* Upload Preferences
* Metadata Profile

---

## Upload Job

Represents a scheduled upload task.

Contains:

* Content Package
* Target Channel
* Schedule
* Upload Status

---

## Asset Library

Asset Library is a supporting component.

It stores reusable assets used during content production.

Examples:

* Footage Images
* Audio Assets
* Prompt Packs
* Thumbnail References

Asset Library is NOT the primary business entity.

The primary business entity is always Content Package.

---

# External Systems

## 9Router AI Infrastructure

Produces:

* Footage Images
* Thumbnail Assets
* Titles
* Descriptions

Content Factory consumes these outputs.

Content Factory does not generate them.
